## Introduction

This documentation aimed to address accessibility cases with existing components in Design System. Documentation based on recommendation [WCAG2.1](https://www.w3.org/TR/WCAG22/) Level AA.
Inspired by [eBay documentation](https://ebay.gitbook.io/mindpatterns), [eBay Bones project](https://github.com/ianmcburnie/bones), [Bootsrtap documentation and components](https://getbootstrap.com/docs/5.0/getting-started/accessibility/).

### Pattern Principles

There are 4 guiding principles of accessibility, collectively know as POUR:

1. Perceivable: People experience content in different ways (sight, hearing, and touch). Content needs to be transferable into recognizable (or perceivable) formats.

2. Operable: Content needs to be navigable (or operable) by multiple methodsâ€”not just a mouse

3. Understandable: Web content needs to be understandable. Language should be simple and concise; functionality should be consistent and intuitive.

4. Robust: Create web content that works for all (or most!) technologies. This includes operating systems, browsers, and mobile devices.
