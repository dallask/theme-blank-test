# Introduction to Storybook

Storybook is a tool for UI development. It makes development faster and easier by isolating components. This allows you to work on one component at a time. You can develop entire UIs without needing to start up a complex dev stack, force certain data into your database, or navigate around your application.

### Benefits

#### For Designers
* Single Source of truth for components
* Access to Design System (Storybook)-the platform with a list of most used components.
* Setup a Single source of truth for new components
* Could be created a separate design system for each brand

#### For Developers
* Single Source of truth for the components
* Low coupled development
* Component quality check on the early stage
* Reusability and maintainability

#### For Brand Owners
* Continuously Developed catalogue of components with the support of options and versioning
* Possibility to easily reuse similar components across projects and across brand
* Possibility to provide early feedback
* Improve collaboration

Use Storybook to build small atomic components and complex pages in your web application. If it's a UI, you can build it with Storybook.

[<img src="/docs/intro/storybook-relationship.png" width="80%"/>](/docs/intro/storybook-relationship.png)

Storybook helps you document components for reuse and automatically visually test your components to prevent bugs. Extend Storybook with an ecosystem of addons that help you do things like fine-tune responsive layouts or verify accessibility.

# Atomic design
**What**

The modern design approach to create components from the simplest one to complex.

* Common components approach though the project

**Benefits**

* Extraction of repeated components parts to “Atoms” reusing between components
* Reducing amount of design discrepancies and communications to resolve them
